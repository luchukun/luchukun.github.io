虚拟机共享宿主机VPN

发表于 2017-02-20 | 分类于 编程

虚拟机无法使用宿主机的VPN网络。
我在workstation player里有台centos虚拟机作为开发机，虚拟机的网络设置为NAT。遇到主机连接了VPN之后，centos无法访问google等被墙的网站，国内网站可以正常访问。
首先我在windows10宿主机上ping了以下www.google.com拿到了google的ip地址。
然后在虚拟机ping了下google的ip地址，发现可以lping通，怀疑是DNS服务器没配置好。
google之后自己实践成功的修改DNS服务器的方法：

1. ifconfig命令查看使用的网卡信息，假设为：****   
`
ifconfig
`  
2. 修改该网卡对应的ifcfg-****文件,添加如下内容:  
`
DNS1="8.8.8.8"
DNS2="192.168.154.2"
DOMAIN="localdomain"
`    
其中DNS1之后为你要设置的DNS服务器地址8.8.8.8是谷歌的dns服务器ip地址:  
`
vim /etc/sysconfig/network-scripts/ifcfg-****  
`
3. 查看/etc/resolv.conf是否和我们设置的DNS内容一致，如果没有我们添加的DNS服务器尝试重启网络服务:  
`
vim /etc/resolv.conf
service network restart
`

之后便可以尝试再次ping www.google.com.
